## What this project is

Salty Dragons is a static HTML site generator. Markdown source files are converted to styled HTML via Pandoc and assembled by a Nix+Ninja build system. The output is deployed to GitHub Pages.

## Build commands

```bash
make all          # Build the site (runs: nix develop -c ninja)
make clean        # Remove outputs/ and build artifacts
make install      # Copy outputs/out/* to $(INSTALLDIR) (set in Makefile.local)
ninja             # Build incrementally (inside nix develop shell)
nix develop       # Enter the dev shell (provides ninja, pandoc, zsh)
nix build -Lv     # Full hermetic Nix build (used in CI)
```

`build.ninja` is auto-generated by `package.nix` — edit the Nix files, not `build.ninja` directly.

## Architecture

### Build pipeline

1. **`package.nix`** — main build derivation; calls `uiop.nix` to flatten all site areas into a page list, collect assets, and emit `build.ninja`, then runs `ninja`.
2. **`uiop.nix`** — pure Nix utility library. Handles path utilities, Ninja rule generation, title extraction from Markdown (YAML front-matter and ATX headings), page slug derivation, and nav link computation.
3. **`build.ninja`** — generated output; defines one `pandoc` rule per page and `copyfile` rules for assets. Do not edit by hand.
4. **`buildPage.zsh`** — invoked per page by Ninja. Receives `title`, `css`, `filter`, `include` (nav HTML), `name`, `prefix` as env vars and runs `pandoc --lua-filter --section-divs --standalone`.
5. **`assets/filter.lua`** — Pandoc Lua filter scaffold (currently a no-op). Add filter functions here to transform AST nodes.
6. **`flake.nix`** — exposes `packages.default` (the site) and the dev shell.

### Content areas

Currently a single area: `home/`.

Each area contains:
- **`default.nix`** — lists pages with metadata (title, CSS, order). This is the file to edit when adding/reordering pages.
- Markdown source files
- An `index-template.md` used to generate the area's index page (where applicable)

`home/` is special: it has no URL prefix. Any other area uses its directory name as prefix.

To add a new area: create a subdirectory with a `default.nix` and add it to the `siteareas` list in `package.nix`.

### Adding a new page

1. Create the Markdown file in the appropriate area subdirectory.
2. Add it to that area's `default.nix` page list.
3. Run `make all` — `build.ninja` regenerates and Ninja builds only the new page.

### Styling

Global CSS lives in `assets/` (`reset.css`, `variables.css`, `nav.css`, `fonts.css`, `figures.css`, `main.css`, `global.css`, `dropcap.css`, `timeline.css`, `index.css`). Per-area CSS is referenced in each area's `default.nix`. Font files (`.woff2`) are copied verbatim to output.
